# ADR-002: Security Headers и CORS
Дата: 2025-12-18
Статус: Accepted

## Context
Текущее FastAPI-приложение "Secure Team Voting Board" не устанавливает защитные HTTP-заголовки
и не конфигурирует CORS (Cross-Origin Resource Sharing). Это создаёт следующие риски:

## Decision
 - [ ] Создать слой обработки middleware для добавления защитных заголовков.
 - [ ] Настроить CORS.
 - [ ] Настроить HSTS для production-окружения.
 - [ ] Написать тесты

## Consequences
**Плюсы:** соответствие NFR-08, R-04, R-07, возможность конфигурировать политики CORS, и среды(разработка, релиз и тд)

**Минусы:** требует написание кода, увеличивает время отклика

## Links
- **NFR-08** (Personal data privacy): ограничение CORS уменьшает риск утечки данных через сторонние домены.
- **R-04** из RISKS.md: перехват данных — HSTS и редирект на HTTPS снижают риск MITM.
- **R-07** из RISKS.md: утечка внутренней информации — строгие заголовки уменьшают поверхность атаки.
